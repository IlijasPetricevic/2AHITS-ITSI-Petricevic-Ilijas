# Übung 3.5 (file)
Es gibt ein Tool mit dem Namen file – recherchiere zur Anwendung.
Diese Datei mit dem Namen qwert enthält eine Textdatei – finde mit Hilfe des file Tools Zugang zu diesem Text.

## Lösung

file bestimmt den Typ einer Datei, indem er deren Inhalt (die ersten Bytes) untersucht, anstatt sich auf die Dateiendung zu verlassen.

```sh
file qwert.tar
```
Es wird ```qwert.tar: POSIX tar archive (GNU)``` ausgegeben.

# Übung 3.6 (cat)
cat unterstützt 2 unterschiedliche Optionen zum Nummerieren der Zeilen. Welche sind das und wie unterscheiden sich diese im Ergebnis?
Was macht die cat Option -E? Was könnte der Sinn dieser Option sein?
Experimentiere mit den Auswirkungen der -s Option von cat.

## Lösung

cat -n:
Nummeriert **alle** Zeilen der Datei.

cat -s:
Fasst wenn es eine leere Zeile in der Datei gibt alles auf eine Zeile zusammen.

cat -E:
Hängt am Ende jeder Zeile ein $ dran. Nützlich um unsichtbare Fehler zu finden.

# Übung 3.7 (date epoch)
Finde per Recherche heraus was die folgende Zeitdarstellung bedeutet:

```sh
$ date +%s
1639927239
```
Verwende date um diese Zeit in lesbarer Form darzustellen.

## Lösung

Dieser Command gibt die Sekunden an, die seit dem 1.Januar 1970 (Unix-Time) vergangen sind.

```sh
date -d @1639927239
```

Es wird das Datum **Sun Dec 19 03:20:39 PM UTC 2021** ausgegeben.
